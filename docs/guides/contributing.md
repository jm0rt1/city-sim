# Contributing Guide

## Workflow
- Create focused branches; keep changes minimal.
- Write or update docs/specs alongside code.
- Run tests locally using `./test.sh`.

## Coding Standards
- Clear function and module contracts documented in specs.
- Favor determinism and reproducibility.
- Keep logging structured and consistent.

## Commit Messages
- Use imperative mood; reference workstreams/specs when relevant.

## Reviews
- Validate acceptance criteria and invariants.
- Check logs and metrics for regressions.
